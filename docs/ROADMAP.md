# Roadmap

This document outlines the planned features and improvements for ioload.

## Short Term (v1.1)

- [ ] Add screenshot/demo GIF to README
- [ ] Publish to PyPI for easy installation via `pip install ioload`
- [ ] Add unit tests for core functionality
- [ ] Support for exporting data to CSV/JSON
- [ ] Add configuration file support
- [ ] Improve error messages and user feedback

## Medium Term (v1.2)

- [ ] Add support for filtering devices by pattern
- [ ] Historical data export
- [ ] Multiple device comparison view
- [ ] Customizable chart colors
- [ ] Support for remote monitoring (SSH)
- [ ] Add man page
- [ ] Performance optimizations for large device lists

## Long Term (v2.0)

- [ ] Web dashboard option
- [ ] Alerting/notification system
- [ ] Plugin system for custom metrics
- [ ] Integration with monitoring systems (Prometheus, etc.)
- [ ] Windows support (using alternative to iostat)
- [ ] Docker container support
- [ ] Kubernetes integration

## Ideas for Community Input

- [ ] What features would you like to see?
- [ ] What platforms should we support?
- [ ] What integrations would be useful?

Please open an issue to discuss any of these features or suggest new ones!
